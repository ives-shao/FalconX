# FalconX 当前开发计划

> 重要声明：
> `Stage 1` 到 `Stage 4` 形成了服务骨架、最小接口和主调用链边界；当时的外部输入与消息发布仍以 `stub provider / listener` 和日志型 publisher 为主。
> 当前在 `Stage 5` 基础上，owner MySQL schema、Redis、Redisson、ClickHouse、Kafka 客户端以及 wallet 的 Web3 SDK 监听骨架都已切换到真实基础设施或真实框架，owner 服务的数据库访问也已切换为 `MyBatis + XML Mapper`。
> 但 `Stage 6A / 6B / 7` 的业务级外部联调尚未完成：真实链上扫块与交易解析、真实业务 Topic 级联调、Swap/隔夜利息结算和生产态安全配置仍未完成，因此当前系统仍不可视为生产可用。

## 1. 计划目标

FalconX 按“先完善架构，再补充业务代码”的原则实施。

当前计划严格遵循：

- [FalconX v1 GODSA 最终架构方案](../architecture/falconx一期网关-服务-数据库架构方案.md)
- [开发启动手册](./开发启动手册.md)
