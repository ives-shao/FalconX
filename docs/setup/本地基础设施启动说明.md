# 本地基础设施启动说明

## 1. 目标

为 FalconX v1 的 5 个后续服务提供共享基础设施：

- MySQL 8.4
- ClickHouse 25.x
- Redis 8.2
- Kafka 4.2

编排文件：

- [docker-compose.yml](../../docker-compose.yml)

## 2. 启动步骤

1. 启动容器

```bash
docker compose up -d
```

2. 查看状态

```bash
docker compose ps
```

3. 停止容器

```bash
docker compose down
```

4. 如需清理数据卷

```bash
docker compose down -v
```

## 3. 默认端口

- MySQL：`3306`
- ClickHouse HTTP：`8123`
- ClickHouse Native：`9000`
- Redis：`6379`
- Kafka：`9092`

## 4. 默认连接信息

### MySQL

- Host：`localhost`
- Port：`3306`
- Username：`root`
- Password：`root`

### ClickHouse

- HTTP Host：`localhost`
- HTTP Port：`8123`
- Native Host：`localhost`
- Native Port：`9000`
- Username：`default`
- Password：空
- Database：`falconx_market_analytics`

### Redis

- Host：`localhost`
- Port：`6379`

### Kafka

- Bootstrap Servers：`localhost:9092`

## 5. 建议创建的 schema

后续开始接数据库时，建议初始化：

- `falconx_identity`
- `falconx_market`
- `falconx_trading`
- `falconx_wallet`
- `falconx_market_analytics`（ClickHouse）

标准运行库与测试库命名说明见：

- [数据库使用说明](./数据库使用说明.md)

## 6. 当前验证方式

当前仓库还是文档阶段，最小验证顺序为：

1. `docker compose config`
2. `docker compose up -d`
3. `docker compose ps`

如果本机没有 Docker daemon，可至少执行第 1 步检查语法。
