# ADR-0003 市场与实时计算边界

## 状态

已采纳

## 背景

FalconX 同时存在两类“看起来都跟价格有关”的能力：

- 行情接入、标准化、K 线生成
- 基于价格的订单触发、盈亏计算、强平判定

如果这两类能力放在同一个服务，会让 `market` 和 `trading` 的边界变模糊。

## 决策

一期固定采用下面的职责划分：

- `market-service` 负责报价与 K 线
- `trading-core-service` 负责基于报价的订单与风险实时计算

## 结果

`market-service` 持有：

- 外部报价接入
- tick 标准化
- `bid / ask / mid / mark`
- Redis 最新价
- ClickHouse `quote_tick / kline`
- 价格与 K 线事件

`trading-core-service` 持有：

- `quote-driven-engine`
- 限价单和止损触发
- 浮盈浮亏刷新
- 保证金率计算
- 强平执行

## 后果

优点：

- `market` 只处理市场数据
- `trading-core` 只处理交易状态和风险状态
- 两边边界清晰，可演进性更好

代价：

- `trading-core` 需要依赖事件和缓存拿到最新价格

## 后续约束

- 不建立独立“实时计算服务”作为一期默认方案
- 若二期要拆 `quote-driven-engine`，必须单独新增 ADR
