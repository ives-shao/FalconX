# FalconX 规则速查表

> 本文件是辅助导航页，不是正式规范来源。若与 `AGENTS.md`、专题规范或 [AI协作与提示规范](./AI协作与提示规范.md) 冲突，以正式规范为准。

## 1. 怎么使用

1. 先判断任务属于哪一类。
2. 再跳到对应正式规范。
3. 如果任务触及契约、schema、事件、状态机或跨服务边界，停止走“快速判断”，直接回到正式规范全文。

“快速通道”只代表执行方式可以更直接，不代表可以跳过正式边界。

## 3. 什么时候不能只看速查表

以下任一出现时，必须回到正式规范，不要只依赖本页：

- 修改 API 路径、方法、字段、响应结构
- 修改数据库 schema、索引、migration 策略
- 修改 Kafka topic、payload、分区键、消费语义
- 修改错误码、状态码、状态机规则
- 修改服务边界、owner 边界、包结构
- 新增跨服务共享模型、缓存快照或共享 DTO
- 改动阶段验收口径、上线表述或生产能力声明

## 4. 最短自检清单

提交前至少确认：

- 是否读了与本任务直接相关的正式规范
- 是否越过了用户未授权的契约变更
- 是否同步更新了必要文档
- 是否按改动范围完成验证，并说明验证结果
- 若修改了仓库内容，是否形成独立 Git 回滚点

更完整的交付标准看 [完成定义](./完成定义.md)。
