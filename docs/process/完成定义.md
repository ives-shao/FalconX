# FalconX v1 完成定义

## 1. 目的

本文件用于定义 FalconX 在实施阶段中“一个任务什么情况下才算完成”。

它适用于：

- 功能开发
- 接口开发
- 事件开发
- 数据库变更
- 文档修订

## 2. 通用完成条件

任一任务要标记为完成，至少满足：

1. 实现符合当前架构与 owner 边界。
2. 代码带有详细注释。
3. 关键业务节点具备日志。
4. 对应测试已补齐并通过。
5. 涉及文档的变更已同步完成。
6. 输出结果与文档一致。
7. 未擅自扩展用户未授权的范围。
8. 若本轮修改了仓库内容，已形成可回滚的 Git 提交点；若用户要求同步远程，已推送到 `GitHub` 远程仓库。
9. 若本轮属于常规开发任务，交付 commit 已落在 `dev` 分支；若涉及 `main`，已得到用户明确授权。

补充要求：

- 若当前实现仍基于 `In-Memory Repository`、`stub provider / listener` 或日志型 publisher，必须在结论和计划文档中显式说明。
- 不允许把“骨架级完成”表述成“真实基础设施已接入”或“可用于生产”。
- 若任务源自问题清单、审计文档或补充事项文档，必须先完成“问题是否真实存在”的核对，再进入修复。
- 若工作区存在与本轮无关的未提交修改，必须先隔离，再为当前任务生成独立回滚点。
- 若任务未要求处理其他分支，实施前默认应在 `dev` 分支开展。

## 2.1 阶段与计划任务完成条件

若本次任务会影响阶段状态或实施顺序，除满足通用完成条件外，还必须满足：

1. [当前开发计划](../setup/当前开发计划.md) 已同步更新。
2. 已明确写出本阶段当前属于“骨架 / stub”还是“真实基础设施接入”状态。
3. 阶段验收同时包含：
   - 业务链路验证结论
   - 对应命令或测试项
4. 已写明下一阶段范围，不能留下计划真空。
5. 若阶段内引入 Redis 快照、Kafka 事件或跨服务共享 JSON 结构，必须写明 TTL / 刷新 / cache miss 策略与兼容性结论。

## 3. 接口任务完成条件

接口任务除满足通用完成条件外，还必须满足：

1. 接口已在目标环境完成开发验证。
2. 请求头、请求参数、响应结构、错误码已固定。
3. 日志点已按规范落位。
4. 已更新统一接口文档：
   - [FalconX统一接口文档](../api/FalconX统一接口文档.md)

以下任一未完成，接口任务不得标记完成：

- 未更新统一接口文档
- 未给出请求/响应示例
- 未给出测试结论

## 4. 数据库任务完成条件

数据库相关任务除满足通用完成条件外，还必须满足：

1. 数据库设计文档已更新。
2. SQL 蓝图已更新。
3. owner 边界未被破坏。
4. 若进入实现阶段，Flyway migration 目录与命名规则已同步落位。
5. 若修改的是作为“基线蓝图”的原始 SQL 文件，同时必须确认运行时 migration 或测试数据库初始化没有出现语义漂移。
6. 若修改的是原始 `V1 / V2` 这类 Flyway 基线脚本，必须同时处理 checksum 影响；验证时应使用新的隔离测试库，或在得到明确授权后执行 repair，不能继续复用已应用旧 checksum 的测试库。

## 5. 事件任务完成条件

事件相关任务除满足通用完成条件外，还必须满足：

1. Topic 命名已纳入 Kafka 事件规范。
2. payload 已定义。
3. 分区键与幂等规则已明确。
4. 发布方和消费方 owner 已明确。
5. 若 payload 或缓存结构新增字段，必须验证消费方兼容性，不得把反序列化风险留到联调阶段。

## 6. 提交前自检清单

提交前至少确认：

- 是否遵守 AGENTS 规则
- 是否更新了必要文档
- 是否补齐测试
- 是否补齐日志
- 是否更新统一接口文档
- 是否说明了验证结果
- 是否有超出用户明确授权范围的内容；若有，是否已经先获得确认
- 若本轮涉及阶段状态变化，是否已同步更新当前开发计划并写明当前实现级别
- 若本轮涉及问题清单修复，是否已回写原问题文档状态
- 若本轮涉及 Redis / Kafka / 跨服务共享 JSON，是否已写明并验证 TTL、刷新、cache miss 和兼容性
- 是否避免并行执行会互相影响的验证命令（例如 `mvn test` 与 `mvn clean compile`）
- 若本轮修改了仓库内容，是否已经生成独立的 Git 回滚点；若用户要求同步远程，是否已经 push 到 `GitHub`
- 若本轮属于常规开发，是否确认提交位于 `dev` 分支；若处理 `main`，是否已得到用户明确授权
