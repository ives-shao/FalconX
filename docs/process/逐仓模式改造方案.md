# 逐仓模式（Isolated Margin）现行口径与剩余范围

## 2. 当前冻结结论

### 2.1 当前交付模式

- 当前只落 `ISOLATED`。
- `CROSS` 只保留字段与错误码预留，不进入当前实现。
- 当前交付仍按 **B-book** 口径推进，不把真实 A-book 对冲纳入本文件范围。

### 2.2 阶段归属口径

- 手动平仓、TP/SL、强平、负净值保护等基础交易闭环能力，属于 `Stage 7` 交易风险闭环事实。
- `marginMode` 显式化、追加逐仓保证金、强平价重算与并发保护，属于 `Stage 7A` 首批逐仓增强子范围事实。
- 本文件记录这些“当前已落地事实”，但不单独宣布 `Stage 7 / 7A` 已验收。

### 2.3 固定规则

- `margin_mode` 当前运行时只真实写入 `ISOLATED`。
- 显式传入 `CROSS` 时，当前北向返回 `40010`。
- `close_reason = 4` 代表强平终态，与手动平仓 / TP / SL 并列。
- `t_ledger.biz_type=10` 固定用于 `isolated_margin_supplement`。
- 追加保证金后必须重算 `liquidationPrice`，并在强平执行前二次校验最新 `margin / liquidationPrice`。
- 追加保证金、手动平仓、强平共享同一并发保护原则，统一使用行级锁模板收敛。

## 3. 当前已落地范围

### 3.1 数据与字段

- `t_position.margin_mode` 已作为真实运行字段存在，当前默认值为 `ISOLATED`。
- `t_position` 相关平仓字段已真实落地并进入写路径。
- `t_trade.trade_type` 已进入真实落地语义。
- `t_ledger.biz_type=10(isolated_margin_supplement)` 已补齐应用枚举、注释迁移与账务落账。

### 3.2 北向接口

- `POST /api/v1/trading/orders/market`
  - 支持 `marginMode` 北向显式化。
  - 未传默认 `ISOLATED`。
  - 显式 `CROSS` 返回 `40010`。
- `POST /api/v1/trading/positions/{positionId}/margin`
  - 已形成首版追加逐仓保证金入口。
- `POST /api/v1/trading/positions/{positionId}/close`
  - 已形成手动平仓真实写路径。
- `PATCH /api/v1/trading/positions/{positionId}`
  - 已形成 TP/SL 修改入口。

### 3.3 交易与风控写路径

- 追加保证金后的 `liquidationPrice` 已进入真实重算路径。
- 强平执行前会基于最新 DB 持仓进行二次校验。
- 追加保证金、手动平仓、强平之间已补 `SELECT FOR UPDATE` 并发保护。
- 强平执行仍基于当前 `QuoteDrivenEngine` 的报价驱动模板运行。

### 3.4 当前测试事实

- `TradingControllerIntegrationTests`
- `TradingUserQueryControllerIntegrationTests`
- `TradingLiquidationIntegrationTests`

这些测试覆盖首批逐仓增强子范围的关键写路径与回归事实，但不单独替代项目阶段验收结论。

## 7. 后续实施阶段前置条件

本节记录 §5.2 冻结结论在后续实施阶段（`STAGE7A-ISOLATED-02`）必须满足的前置条件，供实施时逐项核对。本计划不执行其中任何 migration、代码或测试补齐，只冻结契约。

### 7.1 `t_liquidation_log.margin_mode` 字段定义

- 字段名：`margin_mode`
- 类型：`TINYINT NOT NULL DEFAULT 2`
- 取值：`1=CROSS`、`2=ISOLATED`，与 `t_position.margin_mode` 对齐
- 迁移方式：在 `falconx-trading-core-service` 的下一个 Flyway migration 文件中新增（具体文件名由实施阶段确定）
- 写路径：强平执行服务在写入 `t_liquidation_log` 时取 `t_position.margin_mode` 同步落盘

### 7.2 `GatewaySupplementMarginE2ETests` 场景契约

- 测试位置：`falconx-gateway/src/test/java/com/falconx/gateway/e2e/GatewaySupplementMarginE2ETests.java`
- 联调边界：`gateway + identity-service + market-service + trading-core-service + wallet-service + Kafka`，与现有 `GatewayLiquidationE2ETests` 保持同一启动模式
- 场景步骤（全部必须通过）：
  1. 登录 → 入金到账（`falconx.trading.deposit.credited` 已消费，账户激活）
  2. 下 `ISOLATED` 市价单开仓，记录初始 `liquidationPrice₀`
  3. `POST /api/v1/trading/positions/{positionId}/margin` 追加保证金
  4. 查询 `liquidationPrice₁` 并断言 `liquidationPrice₁ ≠ liquidationPrice₀`（已按最新 `margin` 重算）
  5. 推入报价到 `liquidationPrice₀` — 断言 **不触发** 强平（旧强平价已失效）
  6. 推入报价到 `liquidationPrice₁` — 断言触发强平，`t_position.close_reason=4`，`t_ledger.biz_type=9(liquidation_pnl)`
- 验证证据：测试日志、结构化观测、账本与持仓终态。
