## 1. 品牌定位

FalconX v1 的视觉基调是“冷静、克制、可信的夜间交易界面”，而不是追求艳丽霓虹或炫技金融感。它服务的是长时间看盘和高频读数场景，因此优先级排序是：

1. 信息可读性高于装饰感。
2. 风险状态可感知高于营销氛围。
3. 层次依靠背景、描边和节奏建立，而不是依赖大面积阴影和饱和色块。
4. 数字稳定性和精度感高于普通 UI 的“轻盈感”。

这意味着 FalconX 的深色不是简单把浅色界面反相，而是围绕“减少视觉疲劳、保持长时间扫描效率、让涨跌和风险一眼区分”来建立。

## 2. 视觉原则

### 2.1 长时间看盘友好

- 最底背景 `--bg-0` 禁止纯黑，使用略暖的深蓝黑作为 OLED 友好底色。
- 卡片与浮层以层级背景和 1px 描边构建结构，不在普通卡片上滥用阴影。
- 真正使用阴影的对象只限 `Modal / Drawer / BottomSheet / Toast` 等脱离主平面的浮层。

### 2.2 合约语义优先

- 用户状态、下单态、stale、风控级别、错误码反馈全部以正式契约命名为先，不擅自自造颜色语义。
- TP/SL 属于持仓级语义，因此所有视觉入口都应落在持仓上下文中。
- 一期市价单是同步结果，所以交互态只允许短 loading，不允许做排队或“等待撮合”的伪时延表达。

### 2.3 涨跌必须可读且可解释

- 涨跌禁止只靠颜色表达，必须叠加方向符号或正负号。
- `stale` 和“跌”不能混淆，前者属于数据质量提醒，后者属于行情方向。
- 浮盈亏、百分比、K 线蜡烛、迷你走势都必须遵守同一套涨跌语义。

### 2.4 数字必须稳定

- 价格、数量、PnL、保证金、百分比一律使用数字专用字体。
- 所有关键数值强制启用 `tabular-nums`，避免连续 tick 时列宽抖动。
- 价格更新的动效只能短闪，不允许布局跳动。

## 3. Token 架构

### 3.1 三层结构

FalconX 的 `tokens.json` 使用三层结构：

- `global`：原始设计原子。包括原始色板、字号、间距、圆角、阴影、动效时长。
- `alias`：语义层。把“原始值”翻译成“背景、文字、品牌、上涨、下跌、风险、状态”。
- `component`：组件层。把语义层绑定到具体组件实例，例如按钮、持仓行、OrderForm、QuoteStalenessBadge。

这样做的原因是：

- 调整主题时优先改 `global` 或 `alias`，不用逐个扫组件。
- 原型和未来工程实现都可以直接消费 `component` 级 token。
- 不同页面出现同一种组件时，视觉不会因为页面作者各自发挥而漂移。

### 3.2 命名示例

- `global.color.neutral.950`
- `alias.color.bg.1`
- `component.button.primary.bg.default`
- `component.position.pnl.up`

### 3.3 文件落地规则

- `docs/design/tokens.json` 是主数据源。
- `prototype/assets/styles/tokens.css` 是静态原型可直接使用的 CSS 变量映射。
- CSS 文件中的短名变量如 `--bg-0`、`--up`、`--font-numeric` 都是从 `alias` 层导出的便捷别名，不改变主命名空间。

## 4. 色彩系统

### 4.1 深色底色与层级约束

#### 层级背景

| 语义 | Token | 色值 | 用途 |
|---|---|---:|---|
| 最底层背景 | `alias.color.bg.0` / `--bg-0` | `#07080C` | App 画布、页面底色 |
| 主卡片背景 | `alias.color.bg.1` / `--bg-1` | `#10141B` | 常规卡片、交易终端主容器 |
| 次级浮层背景 | `alias.color.bg.2` / `--bg-2` | `#171D28` | Tab 面板、下拉层、内层模块 |
| 交互面背景 | `alias.color.bg.3` / `--bg-3` | `#202838` | 输入框、hover、选中轨道、骨架亮面 |

#### 设计约束

- `--bg-0` 比纯黑更暖，避免 OLED 纯黑对比过高导致疲劳。
- 相邻背景层的亮度差不得小于 `3%`。本方案采用 RGB 平均亮度近似校验：
  - `bg-0 -> bg-1`：`4.71%`
  - `bg-1 -> bg-2`：`3.79%`
  - `bg-2 -> bg-3`：`4.71%`
- 普通卡片只用 `background + 1px border` 建立层级。
- `Modal / Drawer / BottomSheet / Toast` 才允许叠加阴影。

### 4.2 描边与文本

| 语义 | Token | 色值 | 用途 |
|---|---|---:|---|
| 弱描边 | `alias.color.border.subtle` | `#2A3446` | 常规卡片边界、网格线 |
| 常规描边 | `alias.color.border.regular` | `#3A465D` | 输入框、浮层边界 |
| 强描边 | `alias.color.border.strong` | `#4A5873` | 强调选中、分组区隔 |
| 主文字 | `alias.color.text.primary` | `#E7ECF5` | 主标题、关键数据 |
| 次文字 | `alias.color.text.secondary` | `#AAB4C8` | 标签、说明、辅助数据 |
| 三级文字 | `alias.color.text.tertiary` | `#7A859C` | 占位、弱提示 |
| 禁用文字 | `alias.color.text.disabled` | `#4A5873` | disabled 文案 |

### 4.3 品牌与状态色

| 语义 | Token | 色值 | 用途 |
|---|---|---:|---|
| 品牌主色 | `alias.color.brand.primary` | `#5B8CFF` | 主按钮、焦点、当前激活态 |
| 品牌 hover | `alias.color.brand.hover` | `#7AA4FF` | 主按钮 hover、聚焦外环 |
| 品牌柔光 | `alias.color.brand.soft` | `rgba(91, 140, 255, 0.14)` | focus ring 外围、选区底 |
| 警告 | `alias.color.state.warn` | `#F6B24A` | 待确认、注意事项 |
| 信息 | `alias.color.state.info` | `#46C9F5` | 联机信息、辅助提示 |
| 成功 | `alias.color.state.success` | `#4ADE80` | 成功状态、已完成步骤 |
| 危险 | `alias.color.state.danger` | `#FF6B7A` | 致命错误、破坏性操作 |
| stale 灰 | `alias.color.stale` | `#7A859C` | 数据过期、弱连接状态 |

### 4.4 AA 对比度校验

以下对比度基于 W3C relative luminance 公式，对 `--bg-1 (#10141B)` 进行校验：

| 前景色 | 色值 | 对比度 vs `bg-1` | 结论 |
|---|---|---:|---|
| `up` | `#4ADE80` | `10.59:1` | 通过 AA |
| `down` | `#FF6B7A` | `6.71:1` | 通过 AA |
| `stale` | `#7A859C` | `4.98:1` | 通过 AA |
| `brand` | `#5B8CFF` | `5.84:1` | 通过 AA |
| `text.primary` | `#E7ECF5` | `15.03+:1` | 通过 AA |

结论：

- `up / down / stale` 都满足在主卡片背景上的可读性底线。
- `stale` 虽然是灰色，但不能做得过淡，否则既不能通过 AA，也会和 disabled 状态混淆。

### 4.5 禁止搭配

以下组合在 FalconX 中视为违规：

- 纯黑背景 + 纯白大面积文字。
- 纯红纯绿的股票式涨跌色，不做色盲补偿。
- 在 `bg-1` 上用 `text-tertiary` 充当正文。
- 将 `stale` 灰与 `disabled` 灰混为一谈。
- 在普通卡片上用重阴影替代边界。

## 5. 字体与数字规范

### 5.1 字体家族

| 类型 | Token | 值 | 用途 |
|---|---|---|---|
| 正文字体 | `alias.font.family.body` | `"Inter", "PingFang SC", system-ui, sans-serif` | 标题、标签、段落、按钮文案 |
| 数字字体 | `alias.font.family.numeric` | `"JetBrains Mono", "IBM Plex Mono", ui-monospace, monospace` | 价格、数量、PnL、百分比、traceId |

### 5.2 字阶

| 语义 | Token | 值 | 场景 |
|---|---|---:|---|
| Caption | `alias.font.size.caption` | `11px` | 极小标签、时间标记 |
| Label | `alias.font.size.label` | `12px` | Tabs、Tag、表头 |
| Body | `alias.font.size.body` | `14px` | 正文、默认表单 |
| Price | `alias.font.size.price` | `16px` | 关键价格和数字 |
| Section | `alias.font.size.section` | `20px` | 区块标题 |
| Hero | `alias.font.size.hero` | `32px` | 大标题、主净值 |

### 5.3 数字强约束

所有价格、数量、PnL、百分比必须满足：

- 使用 `--font-numeric` 或 `.num` utility。
- 同时加 `font-variant-numeric: tabular-nums`。
- 可选开启 `font-feature-settings: "tnum" on, "ss01" on`。
- 小数位对齐和正负号对齐都以数字字体为准，不允许依赖字偶距碰运气。

`tokens.css` 已封装：

```css
.num {
  font-family: var(--font-numeric);
  font-variant-numeric: tabular-nums;
  font-feature-settings: "tnum" on, "ss01" on;
}
```

### 5.4 行高与字重

| 语义 | Token | 值 |
|---|---|---:|
| 正文行高 | `alias.lineHeight.body` | `1.5` |
| 数字行高 | `alias.lineHeight.numeric` | `1.2` |
| 标题行高 | `alias.lineHeight.heading` | `1.25` |
| 常规字重 | `alias.font.weight.regular` | `400` |
| 中等字重 | `alias.font.weight.medium` | `500` |
| 半粗字重 | `alias.font.weight.semibold` | `600` |
| 粗体 | `alias.font.weight.bold` | `700` |

## 6. 间距、圆角、边框与阴影

### 6.1 间距体系

系统统一使用：

- `2 / 4 / 6 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48`

建议映射：

- `2 / 4 / 6`：细微内间距、数字微调。
- `8 / 12 / 16`：卡片内边距、表单行间距、按钮高度节奏。
- `20 / 24 / 32`：区块间距、模块切换。
- `40 / 48`：页面段落级间距。

### 6.2 圆角

| 语义 | Token | 值 | 场景 |
|---|---|---:|---|
| `sm` | `alias.radius.sm` | `4px` | Tag、小输入 |
| `md` | `alias.radius.md` | `8px` | 常规按钮、表单 |
| `lg` | `alias.radius.lg` | `12px` | 卡片、Tabs 容器 |
| `xl` | `alias.radius.xl` | `16px` | 浮层、Bottom Sheet |
| `pill` | `alias.radius.pill` | `999px` | 状态胶囊、百分比段控件 |

### 6.3 阴影策略

- 普通卡片：不用阴影。
- Modal / Drawer / BottomSheet / Toast：使用 `global.shadow.overlay`。
- 任何面板如果仅靠阴影区分层次，视为设计违规。

## 7. 动效规范

### 7.1 时长与缓动

| 语义 | Token | 值 | 用途 |
|---|---|---|---|
| 快速反馈 | `global.motion.duration.120` | `120ms` | hover、轻微状态切换 |
| 常规切换 | `global.motion.duration.180` | `180ms` | 面板切换、淡入 |
| 价格闪烁 | `global.motion.duration.240` | `240ms` | 价格上跳 / 下跳 flash |
| stale 呼吸 | `global.motion.duration.2400` | `2400ms` | stale badge 呼吸 |
| 默认缓动 | `global.motion.easing.standard` | `cubic-bezier(0.2, 0.8, 0.2, 1)` | 常规 UI 过渡 |
| 快速淡出 | `global.motion.easing.out` | `ease-out` | ticker flash |
| 平滑呼吸 | `global.motion.easing.inOut` | `ease-in-out` | stale breath |

### 7.2 关键动效

#### 价格闪烁

- 动画：`ticker-flash-up` / `ticker-flash-down`
- 时长：`240ms`
- 缓动：`ease-out`
- 作用：价格变化时短暂显示涨/跌底色，再淡出到透明

实现约束：

- 连续 tick 到来前，必须先取消未完成动画。
- 工程上使用 `animation: none` -> 强制 reflow -> 重新赋值类名或动画。
- 否则在高频更新时会出现叠加抖动。

#### stale 呼吸

- 动画：`stale-breath`
- 时长：`2400ms`
- 缓动：`ease-in-out`
- 作用：提醒数据过期，但不制造强烈警报感

实现约束：

- 呼吸仅用于 stale 标识、stale pill、行情头部提醒，不扩散到整个卡片。
- `prefers-reduced-motion: reduce` 下必须禁用。

### 7.3 动效禁忌

- 不在市价单主路径中做“排队成交中”动画。
- 不在大型数据表格中使用永久闪烁。
- 不为普通 hover 加大幅位移动画。
- 不让 modal 之外的常规卡片产生阴影弹跳。

## 8. 涨跌语义使用规范

### 8.1 总原则

- 涨：`--up` + `↑` 或 `+`
- 跌：`--down` + `↓` 或 `-`
- 平：`--text-secondary` + `—`

颜色只是第一信号，方向符号和字重是第二信号。任何只剩颜色、没有符号或正负号的方案都不合格。

### 8.2 场景矩阵

| 场景 | 颜色 | 符号 | 字重 | 备注 |
|---|---|---|---|---|
| 价格数字 | `up / down / text-secondary` | `↑ / ↓ / —` | `600` | 使用 `.num` |
| 百分比 | `up / down / text-secondary` | `+ / - / —` | `600` | 百分号与数字同字体 |
| 迷你走势 SparkLine | `up / down / stale` | 无内嵌符号，旁边需有数值符号 | `500` | 线本身不能独立承担语义 |
| 持仓 PnL | `up / down / text-secondary` | `+ / - / —` | `600` | 与账户币种同行显示 |
| K 线蜡烛 | `up / down` | 蜡烛本体无符号，但图例或浮层必须有涨跌说明 | `500` | 不额外引入第三套红绿方案 |

### 8.3 使用示例

- `↑ 43,251.20`
- `-2.31%`
- `— 0.00`

错误示例：

- 只把数字染成绿色但没有符号。
- 在同一页中出现“绿涨红跌”和“绿跌红涨”两套系统。
- 用 stale 灰表达“持平”又同时表达“数据过期”。

## 9. 组件清单与规范

> 说明：每个组件都按 `Anatomy / Props / API / States / Usage Dos & Don'ts` 四维定义。交易核心组件额外补充“与后端字段的映射”。

### 9.1 通用组件

#### Button

**Anatomy**

- 容器
- 标签文本
- 可选前置图标
- 可选后置图标
- loading spinner

**Props / API**

- `variant`: `primary | secondary | ghost | danger`，默认 `primary`
- `size`: `sm | md | lg`，默认 `md`
- `fullWidth`: `boolean`，默认 `false`
- `loading`: `boolean`，默认 `false`
- `disabled`: `boolean`，默认 `false`
- `icon`: `string | null`，默认 `null`

**States**

- `default`
- `hover`
- `focus-visible`
- `active`
- `disabled`
- `loading`

**Usage Dos & Don'ts**

- Do：页面主动作只保留一个 `primary`
- Do：破坏性动作使用 `danger`
- Don't：在同一行放两个同权重 primary
- Don't：用 ghost 按钮承载关键交易动作

#### Input

**Anatomy**

- 标签
- 输入区
- 占位文本
- 单位后缀
- 辅助文案
- 错误提示

**Props / API**

- `type`: `text | number | password`，默认 `text`
- `value`: `string`，必填
- `placeholder`: `string`，可选
- `suffix`: `string | null`，默认 `null`
- `error`: `string | null`，默认 `null`
- `disabled`: `boolean`，默认 `false`

**States**

- `default`
- `hover`
- `focus`
- `disabled`
- `error`
- `filled`

**Usage Dos & Don'ts**

- Do：数值输入用字符串语义处理
- Do：错误提示直接贴近输入框
- Don't：金额和数量用浏览器默认 number spinner
- Don't：placeholder 代替标签

#### Select

**Anatomy**

- 当前值容器
- 下拉箭头
- 菜单面板
- 菜单项
- 可选搜索区

**Props / API**

- `options`: `Array<{label:string,value:string}>`，必填
- `value`: `string | null`，默认 `null`
- `placeholder`: `string`，可选
- `searchable`: `boolean`，默认 `false`
- `disabled`: `boolean`，默认 `false`

**States**

- `default`
- `hover`
- `focus`
- `open`
- `selected`
- `disabled`
- `empty`

**Usage Dos & Don'ts**

- Do：链选择、周期选择、symbol 分类选择用 Select
- Do：超过 8 项时支持搜索
- Don't：把二元切换做成 Select
- Don't：菜单项里塞过多次级说明

#### Tabs

**Anatomy**

- 轨道容器
- Tab item
- 激活条
- 可选数量徽标

**Props / API**

- `items`: `Array<{label:string,value:string,count?:number}>`，必填
- `value`: `string`，必填
- `stretch`: `boolean`，默认 `false`
- `size`: `sm | md`，默认 `md`

**States**

- `default`
- `hover`
- `active`
- `focus-visible`
- `disabled`

**Usage Dos & Don'ts**

- Do：同层级内容切换才用 Tabs
- Do：交易面板的“市价 / 限价（开发中）”使用 Tabs
- Don't：跨页面主导航用 Tabs 冒充顶部导航
- Don't：Tab 文案超过 8 个字

#### Tag

**Anatomy**

- 胶囊背景
- 文字
- 可选图标

**Props / API**

- `tone`: `neutral | brand | success | warn | danger`，默认 `neutral`
- `size`: `xs | sm`，默认 `sm`
- `icon`: `string | null`，默认 `null`

**States**

- `default`
- `active`
- `disabled`

**Usage Dos & Don'ts**

- Do：表达分类、占位开发态、风险标签
- Do：控制字数在 2-6 字
- Don't：Tag 承载长句
- Don't：Tag 替代按钮

#### Tooltip

**Anatomy**

- 触发器
- 浮层
- 箭头
- 文案

**Props / API**

## 10. 可访问性

### 10.1 对比度

- 所有正文对背景对比度至少 `4.5:1`
- 大字号（`>=18px`）最低 `3:1`
- 本文档已验证 `up / down / stale / brand` 在 `bg-1` 上均通过 AA

### 10.2 聚焦可见性

- 所有交互组件必须具备明显的 `:focus-visible`
- 推荐样式：`2px` 实线 `--brand-primary` + 外围 `--brand-soft`
- `tokens.css` 已提供通用 focus ring

### 10.3 语义不依赖纯颜色

- 涨跌必须有符号补偿
- 错误和警告必须同时有图标或文案
- UserStatusPill 不只靠背景色区分

### 10.4 图标与可读标签

- 所有图标按钮必须有 `aria-label`
- 纯图标的平仓、复制、编辑、帮助按钮必须有屏幕阅读器标签
- Tooltip 触发器需可键盘聚焦

### 10.5 动效降级

- `prefers-reduced-motion: reduce` 下禁用 `ticker-flash-*` 和 `stale-breath`
- 不因为关闭动效而改变信息语义，颜色和文案仍需完整保留

## 11. 与后续阶段的衔接

- Stage D 的 `components.html` 必须直接消费本文件和 `tokens.css`。
- 后续原型中，任何颜色、圆角、边框、字体、风险色若偏离本文件，都应视为 bug，而不是视觉微调。
- 若后端未来补充订单簿、地址申请、平仓、历史查询等正式契约，应优先更新接口文档，再回写本文件中的“与后端字段的映射”小节。
